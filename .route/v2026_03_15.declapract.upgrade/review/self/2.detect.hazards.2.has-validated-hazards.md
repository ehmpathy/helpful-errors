## confirmed hazards (with evidence)

### HelpfulError polymorphism broken

**evidence:** 7 test failures across 5 files

the test failures prove the hazard is real:

```
src/HelpfulError.test.ts:163
  expect(error).toBeInstanceOf(CustomError);
  Expected constructor: CustomError
  Received constructor: HelpfulError
```

the change from `new this(message, metadata)` to `new HelpfulError(message, metadata)` in static methods `.throw()` and `.wrap()` breaks polymorphism.

**why this matters:**
- consumers rely on `instanceof SubClass` checks
- the `.throw()` method is documented to preserve subclass type
- all error subclasses (BadRequestError, UnexpectedCodePathError, etc.) are affected

**action:** must revert src/HelpfulError.ts lines 190 and 241
