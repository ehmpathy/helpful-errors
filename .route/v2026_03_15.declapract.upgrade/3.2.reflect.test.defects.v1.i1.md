# 3.2.reflect.test.defects - defect classification

## defect: HelpfulError polymorphism broken

### classification: repo quirk

### analysis

**the defect:**
declapract changed `new this(...)` to `new HelpfulError(...)` in static methods, which broke subclass polymorphism.

**why repo quirk:**

1. **HelpfulError uses intentional polymorphism**: the `this` in static methods refers to the caller class constructor, which enables subclasses like `BadRequestError.throw()` to return `BadRequestError` instances.

2. **the practice already acknowledges this use case**: the biome.jsonc comment says:
   ```jsonc
   "noThisInStatic": {
     "level": "warn", // warn w/o autofix, since some usecases like polymorphism require `this` in static methods
     "fix": "none"
   }
   ```

3. **the rule change was correct**: prior config had `"noThisInStatic": "off"`, new config has `"level": "warn", "fix": "none"`. the warn is appropriate to flag potential issues, and `"fix": "none"` prevents auto-apply.

4. **the code change was manual**: since `"fix": "none"` disables autofix, the change to HelpfulError.ts likely came from a manual `npm run fix:lint` or similar as part of the upgrade process.

**why not practice bug:**

the practice correctly sets `"fix": "none"` to prevent auto-application. the comment even documents why: "some usecases like polymorphism require `this` in static methods".

**why not adoption candidate:**

polymorphic static methods that use `this` are rare. most codebases don't have this pattern. this repo is special because HelpfulError is designed for subclass use with `.throw()` and `.wrap()` polymorphism.

---

## recommendation

**for this repo:**
- revert was correct (already done)
- consider a biome suppression comment to document the intentional use:
  ```ts
  // biome-ignore complexity/noThisInStatic: polymorphic subclass instantiation
  throw new this(message, metadata) as InstanceType<T>;
  ```

**for declapract upgrade process:**
- avoid `npm run fix:lint` on upgrades when rules are set to `"fix": "none"`
- let biome warn without auto-apply
