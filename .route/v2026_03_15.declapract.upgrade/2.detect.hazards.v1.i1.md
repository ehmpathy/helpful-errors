# 2.detect.hazards - hazard analysis

## critical hazards

### source code: HelpfulError polymorphism broken

**what changed:**
```ts
// before
throw new this(message, metadata) as InstanceType<T>;

// after
throw new HelpfulError(message, metadata) as InstanceType<T>;
```

**why it may break:**
- subclasses that call `SubClass.throw()` now get `HelpfulError` instead of `SubClass`
- breaks `instanceof` checks for subclass types
- breaks any code that expects specific subclass behavior

**how to verify:**
```sh
npm run test:unit
```

**related:** biome changed `noThisInStatic` from `off` to `warn` which triggered this change. but `this` in static is intentional here for polymorphism.
