# 5.1 execution — registered-symbol brand + version stamp

## approach (locked from vision + wisher)

- brand key: **flat** `Symbol.for('helpful-errors')` (wisher).
- expose `isHelpfulError` **and** `getHelpfulErrorVersion` publicly (wisher).
- version source (Q1): **build-time codegen** `src/version.ts` from `package.json`. verified the
  release flow rewrites only `package.json` (`.github/actions/please-release/action.yml:319`), and
  publish runs `npm run build` (`.publish-npm.yml:57`), so a build-step codegen keeps `dist` correct.
  committed `src/version.ts` seeds tests/typecheck (CI `prepare` skips, so the file must exist).
- semver compare: hand-rolled zero-dep `isVersionAtLeast` (honors no-barrel import discipline).

## peer-review repairs (iteration i004)

- **r8 behavior-intent-coverage blocker.1 (codegen imports its own output) — fixed at root**: the
  i002 r2 fix had `bin/gen.version.ts` import `ConstraintError`, which routes through
  `HelpfulError → ./version` — the very file the tool generates. reverted to a **self-contained**
  codegen: no `src/` import, fail-loud via a clear stderr message + `process.exit(2)` (constraint =
  developer must fix), per `rule.require.exit-code-semantics`. this also resolves the direct r2↔r8
  contradiction (r2 wanted a proper error class; r8 flagged that class's import as a bootstrap
  cycle) — a codegen tool must not depend on its own output. proven: `npm run build:version` →
  `🔖 gen.version → src/version.ts = 1.7.3`.
- **r9 ergo-friction-hazards blocker.1 (unguarded options.version.min) — fixed**: an untyped (js)
  caller could pass `{ version: {} }` or a non-string `min`, a crash path in `isVersionAtLeast`.
  hardened `isHelpfulError` to return **false** (fails gate, no throw) when `min` is not a string —
  honors the vision's "boundary-safe, never crash" contract (a throw would contradict it). added 2
  tests (untyped-omit-min, non-string-min) with documented boundary casts. unit now 154 passed.
- **r8 blocker.2 (build:version wire-up "absent") — non-issue (review-scope false positive)**: the
  wire-up IS present at `package.json:39-40` (`"build:version"` + wired into `build`). the review
  scope is `**/*.{ts,sh}`, which excludes `package.json`, so the reviewer cannot see the proof and
  wrongly infers absence. not a real defect; no code change warranted.
