# blueprint: ConstraintError & MalfunctionError

## summary

add two new error classes to the helpful-errors package:

1. **ConstraintError** — extends BadRequestError, for caller violations
2. **MalfunctionError** — extends UnexpectedCodePathError, for system defects

also:
- add `exit?: number` to HelpfulErrorCode type
- add `static emoji` property to error classes
- fix dynamic prefix for subclasses via `this.constructor.name`
- update README with documentation
